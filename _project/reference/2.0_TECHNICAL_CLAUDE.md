### Button Glow Effects

Animated rainbow glow effects available for buttons (defined in `src/assets/css/components/buttons.css`):

**Edge/Ring Glow** (`has-glow`):
- Adds animated rainbow border glow on hover
- Works on ANY background (solid, transparent, or semi-transparent)
- Uses CSS masking to show only the ring, not the full background
- Default shape is pill (border-radius: 100px)
- Usage: `class="btn btn-primary has-glow"`

**Full Background Glow** (`has-glow-full`):
- Glow fills entire button area on hover (no ring mask)
- Best for buttons where you want the full glow effect visible
- Must be combined with `has-glow`
- Usage: `class="btn btn-dark has-glow has-glow-full"`

**Rectangular Glow** (`has-glow-rect`):
- For elements with slight border-radius (not pill-shaped)
- Use with nav dropdown links, cards, or rectangular buttons
- Must be combined with `has-glow`
- Usage: `class="has-glow has-glow-rect"`

All effects use `--glow-gradient` variable and `gradientMove` animation.

### Video Modal Component

Reusable video modal for Vimeo/YouTube embeds (defined in `src/assets/css/components/video-modal.css` and `src/assets/js/components/video-modal.js`).

**Usage:**

1. Import and initialize in your page JS:
```javascript

}
```

2. Add trigger button with `data-video-modal` attribute:
```html
<a href="#" class="btn-secondary" data-video-modal="yourModalId">
    Watch Video
</a>
```

3. Add modal HTML (before `{% endblock %}`):
```html
<div class="video-modal-overlay" id="yourModalId">
    <div class="video-modal">

                <line x1="18" y1="6" x2="6" y2="18"/>
                <line x1="6" y1="6" x2="18" y2="18"/>
            </svg>

            <iframe src="https://player.vimeo.com/video/YOUR_VIDEO_ID"
                    allow="autoplay; fullscreen; picture-in-picture"
                    title="Video Title"></iframe>
        </div>
```

**Features:**

### JavaScript Patterns

All interactivity uses vanilla JavaScript with these patterns:

**1. Scroll-based Animations:**
```javascript
const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
        if (entry.isIntersecting) {
            entry.target.classList.add('visible');
        }
    });
}, { threshold: 0.1 });
```

**2. Animated Counters:**
- Uses `requestAnimationFrame` for smooth counting
- Easing functions for natural motion
- Triggered by Intersection Observer

**3. 3D Card Tilt Effects:**
- Mouse position tracking with `mousemove`
- `transform: perspective()` for 3D depth
- Parallax effects on card layers

**4. Navigation Behavior:**
- Scroll-aware nav (adds `.scrolled` class)
- Dropdown menus on hover
- Fixed positioning with backdrop blur

**5. Performance Optimizations:**
- `requestAnimationFrame` for smooth animations
- Throttling scroll events with `ticking` flag
- Intersection Observer to trigger only when visible

```javascript
import { identifyUser, trackEvent } from '../components/customerio.js';

// Identify a user (creates/updates profile in Customer.io)
identifyUser({
    email: 'user@example.com',
```

## Making Changes (Modernized System)

### Template System (CRITICAL)

**All pages MUST use this exact pattern:**

```njk
---
title: Page Title | SlashExperts
description: Meta description for SEO
pageCSS: page-slug
pageJS: page-slug
lightNav: false  # Set to true for light navigation variant
---

{% extends "layouts/base.njk" %}

{% block content %}
<!-- Page content here -->
{% endblock %}
```

**IMPORTANT RULES:**
- ✅ **DO** use `{% extends "layouts/base.njk" %}`
- ✅ **DO** use `{% block content %}` and `{% endblock %}`
- ❌ **NEVER** use `layout: layouts/base.njk` in front matter with block tags
- ❌ **NEVER** include `<nav>` or `<footer>` in page content (handled by base.njk)
- ❌ **NEVER** mix the two template inheritance systems

**Why this matters:** Mixing `layout:` front matter with `{% block %}` tags causes silent template rendering failures. All content will disappear from the page.

The navigation automatically switches between:

### Accessibility Standards

**File:** [docs/accessibility/accessibility-standards.md](./docs/accessibility/accessibility-standards.md)

**Component Requirements:**
- Navigation: ARIA attributes, keyboard navigation, focus management
- Buttons: Semantic HTML (`<button>` vs `<a>`), ARIA labels
- Forms: Labels, instructions, error handling
- Accordions: `aria-expanded`, `aria-controls`, single-active pattern
- Tabs: `role="tablist"`, arrow key navigation
- Modals: Focus trapping, Esc to close
- Images: Alt text, decorative vs informative

**Color Contrast Standards:**
- Normal text: 4.5:1 minimum
- Large text (≥24px or ≥19px bold): 3:1 minimum
- UI components: 3:1 minimum
